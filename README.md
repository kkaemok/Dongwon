리뱅 온라인 소속 서버인 **동원 RPG 서버**의 코어 플러그인입니다.

# Dongwon

`Dongwon`은 Paper 기반 RPG 서버 운영에 필요한 메뉴, 직업, 성장, 길드, 파티, 텔레포트, 토지 보호, 개인 창고 및 운영자 도구를 하나의 플러그인으로 제공하는 서버 코어입니다. 플레이어 데이터와 운영 설정을 분리해 장기 운영과 기능 확장을 고려한 구조로 구성되어 있습니다.

## 주요 기능

### 서버 메뉴 및 플레이어 설정

- 서버 메뉴 컴퍼스로 주요 기능 GUI에 접근할 수 있습니다.
- 메인 메뉴에서 텔레포트, 길드, 파티, 송금, 기본 직업 선택, 창고, 땅 설정, 내 정보를 이용할 수 있습니다.
- 서버 메뉴 아이템은 버릴 수 없으며, 분실 시 `/메뉴지급`으로 다시 받을 수 있습니다.
- `/설정` GUI에서 TPA 확인 GUI 사용 여부를 플레이어별로 전환할 수 있습니다.
- 메뉴 제목, 버튼 재료, 이름, lore, 안내 메시지는 `menu.yml`에서 관리합니다.

### 직업 및 숙련도

- 일반 및 특수 직업을 지원하며 스코어보드의 직업명은 등급 색상을 반영합니다.
- 지원 직업: 광부, 전사, 검사, 낚시꾼, 농부, 버퍼, 힐러, 구미호, 오크, 태양의 사제, 안티디버퍼, 천호, 염제-신농.
- 농부는 작물 수확량 증가 효과를 제공합니다.
- 낚시꾼은 낚시 성공 시 숙련도를 획득하며 숙련도 단계에 따라 희귀 보상 확률이 확장됩니다.
- 검사는 숙련도 진행 후 시현류 또는 진-검사 특화를 선택할 수 있습니다.
- 진-검사의 검기는 다중 투사체와 붉은 파티클 궤적을 사용합니다.
- 버퍼는 주변 플레이어에게 상시 지원 효과를 제공하며, 반경 내 파티원에게 방어 효과를 부여하는 사용형 `홀리 아머`를 제공합니다. 홀리 아머는 동시에 하나만 보유할 수 있습니다.
- 힐러는 일정 주기마다 반경 내 파티원을 회복하고 사용형 `힐` 아이템을 제공합니다.
- 숙련도 메시지는 `mastery.yml`에서 변경할 수 있습니다.

### 레벨 및 보스바

- 최대 레벨은 `50`입니다.
- 적대적 몬스터 처치 시 `2 XP`, 위더 및 엔더 드래곤 처치 시 `50 XP`, 워든 처치 시 `100 XP`를 획득합니다.
- 경험치를 획득하면 현재 진행도를 보스바로 표시합니다.
- 파티원이 몬스터를 처치하면 처치자는 경험치 `100%`를 획득하고, 접속 중인 다른 파티원은 각각 해당 경험치의 `10%`를 획득합니다.
- 소수 경험치는 저장값을 유지하면서 화면에는 소수점 한 자리까지 반올림하여 표시합니다.
- 보스바 문구, 색상, 표시 시간은 `bossbar.yml`에서 관리합니다.

### 길드

- 길드 생성, 삭제, 가입 요청, 초대, 수락, 거절, 탈퇴, 퇴출, 목록 및 정보 조회를 지원합니다.
- 길드 표시는 LuckPerms 그룹과 연동됩니다.
- 길드가 삭제되거나 저장 데이터에서 사라진 경우 플러그인은 관련 LuckPerms 그룹과 플레이어 상속 정보를 정리합니다.
- 길드 설정과 메시지는 `guildconfig.yml`, 실제 길드 데이터는 `guilddata.yml`로 분리합니다.

### 파티

- 파티 생성, 초대, 수락, 거절, 퇴출, 해제, 탈퇴 및 정보 확인을 지원합니다.
- 같은 파티원 간 직접 공격과 투사체 공격은 차단됩니다.
- 파티원은 로그아웃 시 파티에서 자동으로 탈퇴 처리됩니다.
- 파티원 목록은 스코어보드에 표시되며 한 줄당 표시 수를 `party.yml`에서 조정할 수 있습니다.
- 파티 경험치 공유 방식은 처치자의 획득량을 감소시키지 않고 다른 파티원 각각에게 추가 경험치를 제공합니다.

### 텔레포트

- TPA는 요청 전 확인 GUI와 수신자의 수락/거절 GUI를 제공합니다.
- TPA GUI 사용 여부는 플레이어별로 설정할 수 있으며 텍스트 명령도 계속 사용할 수 있습니다.
- RTP는 액션바 카운트다운 후 현재 월드 안의 안전한 무작위 위치로 이동합니다.
- RTP 대기 중 수평 이동 시 취소되며 점프는 취소 조건에서 제외됩니다.
- RTP는 오버월드, 네더, 엔드에서 동작하며 월드별 범위와 위험 블록을 `rtp.yml`에서 설정할 수 있습니다.

### 땅 보호

- 플레이어의 현재 위치를 중심으로 설정된 반경의 땅을 생성합니다. 기본 반경은 `50`블럭입니다.
- 월드 제한 없이 동작하며 서로 겹치는 땅은 생성할 수 없습니다.
- 소유자는 허락 요청 수락, 멤버 추방, 소유권 양도 및 삭제를 수행할 수 있습니다.
- 권한이 없는 플레이어의 블록 설치, 파괴, 상호작용을 보호 구역 내에서 차단합니다.
- 경계는 가시성이 높은 파티클로 표시되며 파티클, 사운드, 타이틀과 메시지는 `land.yml`에서 조정합니다.
- 땅 데이터는 별도 설정 파일이 아닌 플레이어 데이터 파일 `profiles.yml`에 저장됩니다.

### 개인 창고 및 경제

- `/창고` 또는 서버 메뉴에서 플레이어별 6줄 개인 창고 GUI를 엽니다.
- 창고 크기, 제목, 사운드와 메시지는 `storageconfig.yml`에서 관리합니다.
- 실제 수납 아이템은 `storagedata.yml`에 저장합니다.
- 캔, 실버캔, 황금캔 화폐 데이터를 관리하며 `/송금`과 관리자 명령을 제공합니다.

### 스코어보드

- 플레이어 이름, 색상 적용 직업, 레벨 경험치, 길드, 파티원, 보유 화폐와 플레이타임을 표시합니다.
- 파티원 표시가 필요 없는 경우에도 불필요한 빈 줄이 생기지 않도록 동적으로 구성됩니다.
- 제목, 표시 줄, 업데이트 주기는 `fastboard.yml`에서 설정합니다.

## 요구 환경

| 항목 | 요구 사항 |
| --- | --- |
| 서버 | Paper `1.21.11` |
| Java | Java `21` |
| 필수 플러그인 | LuckPerms `5.4` 호환 버전 |
| UI 라이브러리 | FastBoard `2.1.5` (`plugin.yml` libraries로 로드) |
| 빌드 도구 | Gradle Wrapper 포함 |

`LuckPerms`는 길드 그룹 및 표시 연동에 필요하므로 서버에 먼저 설치되어 있어야 합니다.

## 설치 및 빌드

### 서버 설치

1. Java 21을 사용하는 Paper 서버를 준비합니다.
2. LuckPerms를 서버의 `plugins` 폴더에 설치합니다.
3. 이 프로젝트에서 빌드한 `Dongwon-1.0-SNAPSHOT.jar`를 `plugins` 폴더에 넣습니다.
4. 서버를 실행하여 기본 YAML 파일과 데이터 디렉터리를 생성합니다.
5. 필요한 설정을 수정한 뒤 `/동원 리로드`를 실행하거나 서버를 재시작합니다.

### 빌드

Windows:

```powershell
.\gradlew.bat build
```

Linux/macOS:

```bash
./gradlew build
```

배포용 JAR 파일은 다음 경로에 생성됩니다.

```text
build/libs/Dongwon-1.0-SNAPSHOT.jar
```

### 로컬 테스트 서버 실행

```powershell
.\gradlew.bat runServer
```

`runServer` 태스크는 Paper `1.21.11` 테스트 서버와 Java 힙 `2G` 설정을 사용합니다.

## 설정 파일

플러그인이 제공하는 문구 설정은 MiniMessage와 레거시 색상 코드를 지원합니다.

```yaml
# MiniMessage
message: '<#55D6FF><bold>안내</bold> <white>메시지'

# Legacy color code
message: '&b&l안내 &f메시지'
```

| 파일 | 역할 | 주요 설정 |
| --- | --- | --- |
| `config.yml` | 공통 설정 | 공통 메시지, 공용 명령/기본값 |
| `menu.yml` | 서버 메뉴 및 설정 GUI | 메뉴 제목, 버튼 재료/이름/lore, 메뉴 메시지, TPA GUI 기본값 |
| `fastboard.yml` | 스코어보드 | 제목, 출력 줄, 업데이트 주기 |
| `bossbar.yml` | 레벨 보스바 | XP 메시지, 색상, 형태, 노출 시간 |
| `mastery.yml` | 숙련도 | 검사/낚시꾼 숙련도 및 특화 메시지 |
| `party.yml` | 파티 | 파티 메시지, 스코어보드 파티원 줄 구성 |
| `guildconfig.yml` | 길드 구성 | 이름 규칙, LuckPerms 그룹 접두사, 길드 메시지 |
| `rtp.yml` | RTP | 카운트다운, 거리, 월드별 범위, 위험 블록, 메시지 |
| `tpa.yml` | TPA | 요청 만료, 확인/응답 GUI 아이템, 메시지 |
| `land.yml` | 땅 보호 | 기본 반경, 최대 보유량, 경계 파티클, 사운드, 타이틀, 메시지 |
| `storageconfig.yml` | 창고 | GUI 줄 수, 제목, 사운드, 메시지 |

기본 설정은 플러그인 업데이트 후에도 기존 운영 값을 덮어쓰지 않고, 새로 추가된 키만 보완하도록 설계되어 있습니다.

## 데이터 저장 구조

설정 파일과 플레이어/서버 데이터 파일은 목적을 분리합니다. 운영 서버를 백업할 때 다음 런타임 데이터 파일을 함께 보존해야 합니다.

| 파일 | 저장 데이터 |
| --- | --- |
| `profiles.yml` | 플레이어 이름, 직업, 화폐, 레벨/XP, 숙련도, 길드 표시, TPA 개인 설정, 소유 땅과 허가 멤버 |
| `guilddata.yml` | 길드, 리더, 멤버, 초대/요청 및 길드 상태 데이터 |
| `storagedata.yml` | 플레이어 개인 창고 아이템 |
| `homes.yml` | 플레이어 홈 위치 |

이전 데이터 파일이 존재하는 경우 일부 시스템은 신규 구조로 자동 이전합니다.

- `jobs.yml`의 직업 데이터는 플레이어 프로필로 이전됩니다.
- `player-settings.yml`의 TPA GUI 설정은 `profiles.yml`로 이전됩니다.
- `lands.yml` 기반 땅 데이터는 프로필 데이터 구조로 이전됩니다.
- `guilds.yml`은 `guilddata.yml`로 이전됩니다.

## 프로젝트 구조

```text
src/main/java/org/kkaemok/dongwon/
|-- admin/        # 관리자 명령
|-- board/        # FastBoard 스코어보드
|-- bootstrap/    # 명령어 및 리스너 등록
|-- economy/      # 송금
|-- guild/        # 길드와 LuckPerms 연동
|-- home/         # 홈 저장 및 이동
|-- job/          # 직업, 패시브, 직업 이벤트
|-- land/         # 땅 생성과 보호
|-- level/        # 레벨, XP, 보스바
|-- menu/         # 서버 메뉴 GUI
|-- party/        # 파티와 파티 효과
|-- progression/  # 숙련도와 프로필
|-- rtp/          # 랜덤 텔레포트
|-- settings/     # 플레이어 설정 GUI
|-- storage/      # 개인 창고
```

## 운영 가이드

### 설정 변경 적용

설정 파일을 수정한 뒤 관리자 권한으로 다음 명령을 실행합니다.

```text
/동원 리로드
```

### 백업 권장 대상

서버 종료 후 최소 다음 파일을 주기적으로 백업하십시오.

```text
plugins/Dongwon/profiles.yml
plugins/Dongwon/guilddata.yml
plugins/Dongwon/storagedata.yml
plugins/Dongwon/homes.yml
```

운영자가 수정한 모든 `*.yml` 설정 파일도 함께 백업하면 서버 표현과 밸런스 설정을 동일하게 복구할 수 있습니다.

### LuckPerms 연동 주의 사항

- 길드 그룹은 `guildconfig.yml`의 `luckperms-group-prefix` 설정에 따라 생성 및 관리됩니다.
- 저장된 길드가 삭제되면 연동된 그룹과 플레이어 상속 정보 정리 로직이 실행됩니다.
- 운영 중 접두사를 변경하기 전에는 LuckPerms 데이터와 `guilddata.yml`을 백업하는 것을 권장합니다.
